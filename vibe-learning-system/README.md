# Vibe Learning System

**涌现式AI交互学习系统**

一个**"反机械化"**的知识管理与学习环境。系统不强制用户进行预分类，而是通过AI辅助的随机碰撞和项目驱动的上下文重组，让知识结构在互动中自然涌现。

**隐喻**：从"仓库管理员"（存储）转变为"炼金术士"（转化）。

---

## 🎯 核心愿景

### 用户痛点
- **收集癖**：只存不看，稍后读变成"永不读"
- **分类瘫痪**：花费大量时间思考"这个文件该放哪个文件夹"，而非思考内容本身
- **孤岛效应**：知识点之间缺乏联系，无法跨学科解决问题
- **死板检索**：只能通过关键词搜索，无法通过模糊意图或语义关联找到灵感

### 解决方案
**三大核心支柱**：

1. **土壤（输入）**：极低摩擦的信息捕获
2. **炼金（碰撞）**：AI驱动的随机碰撞和关联分析
3. **建造（项目）**：语义检索驱动的项目构建

---

## 🌱 支柱一：无压力的"土壤"（输入）

### 目标
极低摩擦的信息捕获

### 功能
- 支持文本的快速抛入
- **AI自动去噪**：自动提取长文的核心观点，生成1-2句"种子摘要"
- **语义标签**：AI在后台打上语义标签（Vector Embedding），为后续的"吸附"做准备

### 示例
用户输入：
```
熵增定律很有趣，它说宇宙的总熵只会增加不会减少，这让我想到我们的房间如果不收拾就会越来越乱，这是不是一种对抗熵增的努力？
```

AI修剪：
```
01. 清理房间是对抗宇宙熵增的微观努力
```

---

## ⚗️ 支柱二：AI驱动的"炼金"（碰撞）

### 目标
利用AI的联想能力，强制产生连接

### 功能
- **随机碰撞UI**：系统随机展示两个种子卡片
- **AI催化剂（核心）**：当用户点击"寻找联系"时，LLM分析两张看似无关的卡片，并生成3个潜在的关联方向
- **合成新知**：用户基于AI的提示确认连接，生成一张新的"洞察卡片"

### 例子
卡片A（生物进化） + 卡片B（软件开发）
→ AI提示："软件版本的迭代是否遵循优胜劣汰？"

### AI催化剂三要素
1. **隐喻共鸣**：用一个领域解释另一个领域
2. **矛盾冲突**：找出两个概念的矛盾点
3. **因果关系**：探索可能的因果链条

---

## 🏗️ 支柱三：磁性"建造"（项目）

### 目标
通过真实问题（Project）来收敛知识

### 功能
- **项目定义**：用户输入一个模糊的目标（如"我想写一篇关于非线性叙事的文章"）
- **AI磁铁（语义检索）**：系统不只是搜索关键词，而是搜索与"非线性叙事"概念相关的笔记（即使笔记里没有这个词，只有"时间旅行"或"记忆碎片"）。AI自动将相关度高的笔记"吸附"到项目看板周围
- **苏格拉底助手**：在构建过程中，AI会主动提问："你目前的论据似乎缺乏关于'情感节奏'的支持，需要补充这方面的种子吗？"

---

## 🤖 AI角色定义

在这个系统中，AI**不是**：
- ❌ 自动帮你写完论文的代笔者
- ❌ 只是用来聊天的ChatGPT窗口

在这个系统中，AI**是**：
- ✅ **园丁**：帮你修剪杂乱的信息，通过修剪让核心显露
- ✅ **媒人**：向你介绍两个陌生概念，并告诉你它们为什么可能是一对
- ✅ **策展人**：当你设定展览主题（项目）时，它从库房里找出最合适的展品

---

## 📖 交互流程

### 典型流程
1. **输入**：用户丢入一句话："熵增定律很有趣"
2. **处理**：AI后台默默将其向量化
3. **唤醒**：第二天，用户打开"炼金台"。系统推出："熵增定律" + "房间整理"
   - AI提示："你觉得'清理房间'是不是一种对抗宇宙熵增的微观努力？"
4. **连接**：用户受到启发，点击"连接"，写下新笔记："生活是对抗熵增的战斗"
5. **输出**：用户创建项目"生活哲学"，系统自动把这条新笔记吸附过来

## 🏗️ 系统架构

### 文件结构
```
vibe-learning-system/
├── _系统/（系统文件）
│   ├── CLAUDE.md（主入口，动态加载）
│   ├── 输入Agent.md（园丁角色）
│   ├── 炼金Agent.md（媒人角色）
│   ├── 建造Agent.md（策展人角色）
│   └── 规范/
│       └── 核心规范.md（所有Agent遵守的核心规则）
└── 库房/（所有内容）
    ├── 项目信息.md（全局状态）
    ├── 种子池/（所有捕获的种子）
    ├── 洞察卡片/（炼金生成的洞察）
    └── 项目/（所有项目）
        └── [项目名]/
            ├── 项目信息.md
            ├── 吸附白板.canvas
            └── 最终产出.md
```

### Agent职责解耦

| Agent | 角色 | 核心动作 | 更新字段 |
|-------|------|---------|---------|
| 输入Agent | 园丁 | 修剪摘要 + 向量化标记 | 种子状态 |
| 炼金Agent | 媒人 | 随机碰撞 + 关联分析 + 生成洞察 | 炼金记录、洞察卡片列表 |
| 建造Agent | 策展人 | 语义吸附 + 白板组织 + 生成文章 | 建造状态、输出记录 |

### 动态加载机制
- 首次对话：读取`CLAUDE.md`，执行"对话启动检查"
- 根据用户意图，动态加载对应的Agent模块
- 避免一次性加载所有内容，保持轻量级

## 🚀 开始使用

### 首次使用
1. 打开Claude Code，切换到`vibe-learning-system`目录
2. 对话开始时，AI会自动检测工作目录
3. AI会显示欢迎语，引导你开始

### 典型场景
```
用户：我想开始

AI：欢迎来到 Vibe Learning！

这是你的知识炼金台，我是你的园丁、媒人、策展人。

开始吧，抛入你的第一个想法：

用户：熵增定律很有趣，它说宇宙的总熵只会增加不会减少，
      这让我想到我们的房间如果不收拾就会越来越乱，

AI（输入Agent）：

我帮你修剪了核心观点：
清理房间是对抗宇宙熵增的微观努力

继续抛入种子，或者说"进入炼金台"开始碰撞？

用户：进入炼金台

AI（炼金Agent）：
⚗️ 炼金台
```

## 📚 技术原理

### 语义检索（Vector Embedding）
1. **输入阶段**：AI在捕获种子时自动向量化
2. **炼金阶段**：AI在生成洞察时自动向量化
3. **建造阶段**：基于向量相似度检索相关内容
4. **优势**：不只是关键词匹配，而是语义相关

### 随机碰撞机制
- 完全随机抽取，不考虑相关性
- 越看似无关，越能激发创新
- 避免陷入思维定式

### 白板组织
- 可视化思维过程
- 自动识别逻辑链条
- 支持用户手动调整

## 🔄 工作流切换

### 输入 → 炼金
- 捕获3个种子后，AI主动建议进入炼金台
- 用户可以明确说"进入炼金台"

### 炼金 → 建造
- 炼金5个洞察后，AI主动建议创建项目
- 用户可以明确说"创建项目"

### 建造 → 输入
- 项目建造过程中，用户可以随时捕获新种子
- 新种子自动进入种子池，供后续检索吸附

### 任意切换
- 用户可以用快捷命令明确切换
- 不需要按固定流程，可以来回跳转
